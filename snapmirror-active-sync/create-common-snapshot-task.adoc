= Crear una instantánea común entre grupos de consistencia de ONTAP o SVM de NAS
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Además de las operaciones de instantánea programadas de forma regular, puedes crear manualmente una link:../concepts/snapshot-copies-concept.html["snapshot"] común entre los volúmenes del grupo de consistencia primario SnapMirror y los volúmenes del grupo de consistencia secundario SnapMirror en una configuración SAN o a nivel de SVM para configuraciones NAS.

.Acerca de esta tarea
El intervalo programado para la creación de instantáneas es de 12 horas para la política Automated Failover y de 3 horas para la política Automated Failover Duplex.

.Antes de empezar
* La relación de grupo SnapMirror debe estar sincronizada.


.Pasos
Puedes crear una instantánea común usando System Manager o la CLI de ONTAP.

[role="tabbed-block"]
====
.System Manager
--
. Desde el clúster de destino, ve a *Protección > Replicación*.
. Localiza la fuente de la relación, selecciona image:icon_kabob.gif["Icono de opciones de menú"] y elige *Actualizar*.
. Selecciona *Según la política* o *Selecciona la instantánea* para elegir una instantánea ya existente.
. Selecciona *Actualizar*.


--
.CLI
--
. Desde el sitio de destino, crea una instantánea común:
+
`snapmirror update -destination-path <destination path>`

+
Ejemplo de configuración de una SAN:

+
[listing]
----
destination::>snapmirror update -destination-path vs1_dst:/cg/cg_dst
----
+
Ejemplo de configuración de un NAS:

+
[listing]
----
snapmirror update -destination-path vs_b:
----
. Supervise el progreso de la actualización:
+
`destination::>snapmirror show -fields newest-snapshot`

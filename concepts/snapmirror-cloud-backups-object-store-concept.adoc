= SnapMirror copias de seguridad en la nube en almacén de objetos
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
_SnapMirror Cloud_ es una tecnología de copia de seguridad y recuperación diseñada para los usuarios de ONTAP que quieren trasladar sus flujos de trabajo de protección de datos a la nube o a una solución de almacenamiento en almacén de objetos local. Las organizaciones que están dejando atrás las arquitecturas heredadas de copia de seguridad en cinta pueden usar el almacén de objetos como repositorio alternativo para la conservación y el archivado de datos a largo plazo. SnapMirror Cloud ofrece replicación de ONTAP a almacén de objetos como parte de una estrategia de copia de seguridad incremental permanente.

SnapMirror Cloud replication es una función de ONTAP sujeta a licencia. SnapMirror Cloud se introdujo en ONTAP 9.8 como una extensión de la familia de tecnologías de replicación SnapMirror. Aunque SnapMirror se utiliza con frecuencia para copias de seguridad de ONTAP a ONTAP, SnapMirror Cloud usa el mismo motor de replicación para transferir instantáneas de ONTAP a copias de seguridad en almacenamiento de objetos compatible con S3.

Orientado a casos de uso de copias de seguridad, SnapMirror Cloud admite tanto la retención a largo plazo como los flujos de trabajo de archivo. Al igual que con SnapMirror, la copia de seguridad inicial de SnapMirror Cloud realiza una transferencia completa de un volumen. Para las copias de seguridad posteriores, SnapMirror Cloud genera una instantánea del volumen de origen y transfiere la instantánea, incluyendo solo los bloques de datos modificados, a un destino de almacén de objetos.

SnapMirror Cloud permite configurar relaciones entre sistemas ONTAP y ciertos destinos de almacén de objetos locales y en la nube pública, incluyendo Amazon S3, Google Cloud Storage y Microsoft Azure Blob Storage. Otros destinos de almacén de objetos locales incluyen StorageGRID y ONTAP S3.

Además de utilizar ONTAP System Manager para gestionar las configuraciones de SnapMirror Cloud, existen varias opciones de orquestación disponibles para gestionar las copias de seguridad de SnapMirror Cloud:

* Existen varios socios de backup de terceros que ofrecen compatibilidad con la replicación en la nube de SnapMirror. Los proveedores participantes están disponibles en link:https://www.netapp.com/blog/new-backup-architecture-snapdiff-v3/["Blog de NetApp"^].
* NetApp Backup and Recovery para una solución nativa de NetApp para entornos ONTAP
* API para desarrollar software personalizado para los flujos de trabajo de protección de datos o para aprovechar herramientas de automatización



NOTE: La replicación en la nube de SnapMirror que es gestionada por System Manager solo admite StorageGrid y ONTAP S3 como destino de almacenamiento de objetos. No se admiten otros proveedores.


NOTE: El uso de las API de la nube de SnapMirror suele estar restringido a los socios de copias de seguridad. Los clientes que deseen desarrollar sus propias soluciones basadas en API deben ponerse en contacto con su equipo de cuentas de NetApp para obtener información sobre las condiciones de la licencia.

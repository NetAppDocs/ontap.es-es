= Mostrar información sobre los filtros de seguimiento de seguridad en las SVM de ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Consulta los detalles del filtro de rastreo de seguridad en tu máquina virtual de almacenamiento (SVM) para identificar los eventos de acceso que rastrea cada filtro.

[role="tabbed-block"]
====
.System Manager
--
A partir de ONTAP 9.6, puedes usar System Manager para rastrear el acceso a archivos en la SVM.

.Pasos
. Abre el panel de control de System Manager.
. En *Cluster*, selecciona *Storage VMs*.
. Selecciona image:icon_kabob.gif["Icono de opciones de menú"] para la SVM aplicable.
. Selecciona *Rastrear acceso a archivos*.
. Configura el tipo de protocolo de acceso a archivos como *SMB/CIFS* o *NFS*.
. Ingresa el *User name*.
. Introduce la *dirección IP del cliente*.
. Opcionalmente, selecciona una o más de las siguientes:
+
.. Selecciona *Rastrear ruta concreta* para especificar el archivo o la ruta de archivo que quieres rastrear.
.. Selecciona *Mostrar solo entradas de acceso denegado* para limitar el rastreo a los eventos de acceso denegado.


. Selecciona *Iniciar rastreo*.


--
.CLI
--
Usa la CLI para rastrear el acceso a archivos en la SVM.

.Paso
. Muestra información sobre las entradas del filtro de seguimiento de seguridad mediante `vserver security trace filter show` el comando.
+
Obtenga más información sobre `vserver security trace filter show` en el link:https://docs.netapp.com/us-en/ontap-cli/vserver-security-trace-filter-show.html["Referencia de comandos del ONTAP"^].



.Ejemplos
El siguiente comando muestra información sobre todos los filtros de seguimiento de seguridad en la SVM vs1:

[listing]
----
cluster1::> vserver security trace filter show -vserver vs1
Vserver  Index   Client-IP            Path            Trace-Allow  Windows-Name
-------- -----  -----------  ----------------------   -----------  -------------
vs1        1    -            /dir1/dir2/file.txt          yes      -
vs1        2    -            /dir3/dir4/                  no       mydomain\joe
----

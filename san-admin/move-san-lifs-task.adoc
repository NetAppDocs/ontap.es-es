= Mover una LIF SAN de ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Si es necesario desconectar un nodo, puedes mover un SAN LIF para conservar su información de configuración, como su WWPN. Mover un LIF no requiere volver a zonificar tu estructura FC ni crear nuevas sesiones iSCSI entre los hosts del clúster y la interfaz de destino. Puedes mover SAN LIFs a cualquier nodo del clúster, pero no entre máquinas virtuales de almacenamiento (SVM).

Puedes mover un LIF para hacer lo siguiente sin interrupciones:

* Sustituir un par de HA de un clúster por un par de HA actualizado de forma transparente para los hosts que acceden a datos de LUN o namespace
* Actualizar una tarjeta de interfaz de destino
* Traslade los recursos de una máquina virtual de almacenamiento (SVM) de un conjunto de nodos de un clúster a otro conjunto de nodos del clúster


No puedes mover un SAN LIF usando el comando `network interface move`. Para mover un SAN LIF, tienes que ponerlo fuera de línea, moverlo a un nuevo nodo o puerto de origen y luego volver a ponerlo en línea. Como un SAN LIF debe ponerse fuera de línea antes de moverlo, el tráfico del host debe depender del software de multivía del host para proporcionar acceso sin interrupciones al LUN y así evitar cualquier interrupción de E/S.

.Antes de empezar
Si el LIF es miembro de un conjunto de puertos, es necesario haber eliminado el LIF del conjunto de puertos antes de poder mover el LIF a un nodo diferente.

.Acerca de esta tarea
El nodo de destino y el puerto físico de un LIF que desee mover deben estar en la misma estructura de FC o red Ethernet. Si mueve un LIF a otra estructura que no haya tenido una zona adecuada o si mueve un LIF a una red Ethernet que no tenga conectividad entre un iniciador iSCSI y un destino, no será posible acceder a la LUN cuando vuelva a estar en línea.

.Pasos
. Vea el estado administrativo y operativo de la LIF:
+
[source, cli]
----
network interface show -vserver <SVM_name>
----
+
Obtenga más información sobre `network interface show` en el link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["Referencia de comandos del ONTAP"^].

. Cambie el estado de la LIF a `down` (sin conexión):
+
[source, cli]
----
network interface modify -vserver <SVM_name> -lif <LIF_name> -status-admin down
----
+
Obtenga más información sobre `network interface modify` en el link:https://docs.netapp.com/us-en/ontap-cli/network-interface-modify.html["Referencia de comandos del ONTAP"^].

. Asigne a la LIF un nodo y un puerto nuevos:
+
[source, cli]
----
network interface modify -vserver <SVM_name> -lif <LIF_name> -home-node <node_name> -home-port <port_name>
----
. Cambie el estado de la LIF a `up` (en línea):
+
[source, cli]
----
network interface modify -vserver <SVM_name> -lif <LIF_name> -status-admin up
----
. Compruebe los cambios:
+
[source, cli]
----
network interface show -vserver <SVM_name>
----

= Activar o desactivar NFS para los SVM de ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Puedes activar o desactivar las versiones de NFS usando System Manager o la CLI. Esto permite el acceso a archivos para clientes que usan el protocolo NFS.

Por defecto, NFSv3 está activado en todas las versiones compatibles de ONTAP 9. A partir de ONTAP 9.9.1, NFSv4.1 y NFSv4.2 están activados por defecto; en versiones anteriores, están desactivados por defecto.

A partir de ONTAP 9.19.1, puedes activar o desactivar NFSv4.1 y NFSv4.2 de forma independiente. Antes de ONTAP 9.19.1, solo podías activar o desactivar NFSv4.1 y NFSv4.2 juntos. Por defecto, tanto NFSv4.1 como NFSv4.2 están habilitados cuando se crea un nuevo servicio de NFS, a menos que el administrador los desactive explícitamente.


NOTE: No se admite la supervisión de FPolicy de eventos de acceso a archivos NFSv4.2. Si configuras la supervisión de FPolicy, necesitas desactivar NFSv4.2 en la SVM. Para referencia, consulta link:../nas-audit/supported-file-operation-filter-fpolicy-nfsv4-concept.html["Combinaciones de filtros y operaciones de archivos compatibles que ONTAP FPolicy monitorea para NFSv4"].

[role="tabbed-block"]
====
.System Manager
--
.Pasos
. En System Manager, selecciona *Cluster > Storage VMs*, selecciona la VM de almacenamiento, selecciona *Settings* y luego selecciona image:icon_gear.gif["El icono Actions"] o image:icon-edit-pencil-blue-outline.png["Icono Editar"] en *NFS*.
. Selecciona la casilla de comprobación *Habilitar NFS*.
. Activa o desactiva un protocolo NFS:
+
** Para activar NFSv3, selecciona la casilla de comprobación *Activar soporte versión 3* y ten en cuenta la siguiente opción:
+
*** Selecciona la casilla de comprobación *Habilitar identificadores de archivos y FSID de 64 bits* para habilitar la compatibilidad con identificadores de archivos y FSID de 64 bits para NFSv3.


** Para desactivar NFSv3, desactiva la casilla de comprobación *Activar soporte versión 3*.
** Para habilitar NFSv4.0, selecciona la casilla de comprobación *Habilitar versión de compatibilidad 4* y ten en cuenta las siguientes opciones:
+
*** Selecciona la casilla de comprobación *Habilitar ACL* para permitir el uso de una lista de control de acceso.
*** Selecciona la casilla de comprobación *Habilitar delegación de lectura* para permitir el acceso de solo lectura.
*** Selecciona la casilla de comprobación *Habilitar delegación de escritura* para permitir el acceso de escritura.
*** Selecciona la casilla de comprobación *Habilitar identificadores de archivos y FSID de 64 bits* para habilitar la compatibilidad con identificadores de archivos y FSID de 64 bits. Esta casilla de comprobación se aplica a NFSv4.0, NFSv4.1 y NFSv4.2. Esto es independiente de la casilla de comprobación usada para controlar la compatibilidad con 64 bits para NFSv3.


** Para desactivar NFSv4.0, desactiva la casilla de comprobación *Activar soporte versión 4*.
** Para activar NFSv4.1, selecciona la casilla de comprobación *Enable support version 4.1 and 4.2* y luego selecciona la casilla de comprobación *Enable support version 4.1*. Ten en cuenta las siguientes opciones:
+
*** Selecciona la casilla de comprobación *Habilitar ACL* para permitir el uso de una lista de control de acceso.
*** Selecciona la casilla de comprobación *Habilitar delegación de lectura* para permitir el acceso de solo lectura.
*** Selecciona la casilla de comprobación *Habilitar delegación de escritura* para permitir el acceso de escritura.
*** Selecciona la casilla de comprobación *pNFS* para permitir que los clientes utilicen NFS paralelo (pNFS) para mejorar el rendimiento.
*** Selecciona la casilla de comprobación *Habilitar identificadores de archivos y FSID de 64 bits* para habilitar la compatibilidad con identificadores de archivos y FSID de 64 bits. Esta casilla de comprobación se aplica a NFSv4.0, NFSv4.1 y NFSv4.2. Esto es independiente de la casilla de comprobación usada para controlar la compatibilidad con 64 bits para NFSv3.


** Para desactivar NFSv4.1, desactiva la casilla de comprobación *Activar soporte versión 4.1*.
** Para activar NFSv4.2, selecciona la casilla de comprobación *Activar compatibilidad con las versiones 4.1 y 4.2* y luego selecciona la casilla de comprobación *Activar compatibilidad con la versión 4.2*. Considera las siguientes opciones:
+
*** Selecciona la casilla de comprobación *Habilitar ACL* para permitir el uso de una lista de control de acceso.
*** Selecciona la casilla de comprobación *Habilitar delegación de lectura* para permitir el acceso de solo lectura.
*** Selecciona la casilla de comprobación *Habilitar delegación de escritura* para permitir el acceso de escritura.
*** Selecciona la casilla de comprobación *pNFS* para permitir que los clientes utilicen NFS paralelo (pNFS) para mejorar el rendimiento.
*** Selecciona la casilla de comprobación *Habilitar identificadores de archivos y FSID de 64 bits* para habilitar la compatibilidad con identificadores de archivos y FSID de 64 bits. Esta casilla de comprobación se aplica a NFSv4.0, NFSv4.1 y NFSv4.2. Esto es independiente de la casilla de comprobación usada para controlar la compatibilidad con 64 bits para NFSv3.


** Para desactivar NFSv4.2, desactiva la casilla de comprobación *Activar soporte versión 4.2*.


. Opcionalmente, puedes especificar el *nombre de dominio de ID V4* para el protocolo NFS.
. Seleccione *Guardar*.

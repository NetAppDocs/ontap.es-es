= Modifica los grupos de políticas de QoS creados en ONTAP 9.19.1 o posterior antes de revertir desde ONTAP 9.19.1
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Los grupos de políticas de QoS adaptativos y no adaptativos se habilitan automáticamente para aumentar temporalmente el límite máximo de rendimiento de una carga de trabajo en un 50% durante 1 segundo para responder por defecto a demandas de mayor rendimiento a corto plazo. A partir de ONTAP 9.19.1, puedes modificar esta configuración predeterminada. En ONTAP 9.18.1 y versiones anteriores, esta configuración no se puede cambiar. Si has modificado la configuración predeterminada de cualquier grupo de políticas de QoS creado en ONTAP 9.19.1 o posterior, debes restablecerla antes de poder revertir de ONTAP 9.19.1 a ONTAP 9.18.1.

.Pasos
. Verifica si tienes algún grupo de políticas QoS habilitado para el aumento automático del límite de caudal:
+
[source, cli]
----
qos policy-group show – instance
----
. Si algún grupo de políticas está configurado con `burst-percent`, `burst-iops` y `burst-duration`, establece estas opciones en 0.
+
[source, cli]
----
qos policy-group modify -policy-group <policy-group> -burst-percent 0 -burst-iops 0 -burst-duration 0
----

= Asignar acceso de administrador a SVM con ONTAP System Manager
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
A partir de ONTAP 9.19.1, los administradores de clústeres ONTAP pueden asignar acceso a System Manager a la persona que gestiona la SVM, conocida como el administrador de SVM.

Esto es especialmente útil en un entorno ONTAP con inquilinos, donde el administrador del clúster puede proporcionar un panel de control SVM para cada inquilino.

Los administradores de clúster también pueden permitir que sus administradores de SVM creen volúmenes.



== Paso 1: prepara la interfaz de red de gestión para el panel de control de SVM

Antes de asignar el panel de control de la SVM a tu administrador de SVM, asegúrate de que la SVM tiene una interfaz de red de gestión. Si la SVM no tiene una interfaz de red, puedes crear una.

.Pasos
. En System Manager, selecciona *Interfaces de red > Descripción general* y luego selecciona *Add* para crear una interfaz de red.
. Selecciona *Storage VM management* como tipo de interfaz de red.
. Especifica la *dirección IP* y la *máscara de subred* para la interfaz de red de gestión del SVM.




== Paso 2: Verifica la configuración de la cuenta de usuario vsadmin

Siempre que se crea cualquier SVM, ONTAP crea automáticamente una cuenta de usuario vsadmin asociada. Antes de asignar el cuadro de mando de la SVM al administrador, asegúrate de que la cuenta de usuario vsadmin tiene una contraseña y está desbloqueada.

.Pasos
. Abre el panel de control de System Manager como administrador de clúster.
. Selecciona *Cluster > Storage VMs*.
. Selecciona la SVM, selecciona *Configuración* y luego selecciona *Seguridad > Usuarios y roles*.
. Junto al usuario *vsadmin*, selecciona image:icon_kabob.gif["Icono de opciones de menú"] y, si es necesario, selecciona:
+
.. *Cambiar contraseña* para actualizar la contraseña de vsadmin.
.. *Desbloquea* para asegurarte de que es utilizable.






== Paso 3: Permitir a los administradores de SVM añadir volúmenes (opcional)

Como administrador del clúster ONTAP, si quieres que tus administradores de SVM puedan añadir volúmenes locales en sus SVM, puedes concederles permiso.

.Pasos
. Abre el panel de control de System Manager e inicia sesión como administrador del clúster.
. En *Cluster*, selecciona *Storage VMs*.
. Selecciona image:icon_kabob.gif["Icono de opciones de menú"] para la SVM aplicable.
. Seleccione *Editar*.
. En *Asignación de recursos*, selecciona la casilla de comprobación *Limitar la creación de volúmenes a los niveles locales preferidos*.
. De forma predeterminada, se incluyen todos los niveles locales al seleccionar la casilla de comprobación. Puedes eliminar niveles específicos si es necesario. Si eliminas todos los niveles locales o dejas la casilla de comprobación en blanco, los administradores de SVM no pueden aprovisionar volúmenes locales.
. Seleccione *Guardar*.




== Paso 4: Asignar acceso de administrador a SVM

Para asignar el acceso al panel de control de SVM, proporciona la dirección IP de gestión de SVM y la contraseña vsadmin a cada administrador de SVM.

.Pasos
. En System Manager, selecciona *Network interfaces > Overview* y luego selecciona la interfaz de red *Management*.
. Selecciona la dirección IP y compártela, junto con el nombre de usuario y la contraseña *vsadmin*, con el administrador de la SVM.
